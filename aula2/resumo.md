
Resumo — Definição de Escopo, Prototipação e UX

1. Tipos de Aplicações Web

As aplicações web podem ter diferentes objetivos. Entre os principais tipos estão:

* Site institucional:** apresenta a empresa, sua história, produtos e serviços.
* Blog: focado na publicação de conteúdos e notícias.
* E-commerce: utilizado para venda de produtos ou serviços online.
* Landing Page: página criada para campanhas ou conversões específicas.
* Plataforma: sistemas mais completos com várias funcionalidades.

2. Definição de Escopo

O escopo define tudo o que será desenvolvido no projeto. Ele ajuda a estabelecer limites, objetivos e funcionalidades do site ou sistema.

A definição do escopo é importante porque:

* facilita a comunicação entre equipe e cliente;
* organiza o desenvolvimento do projeto;
* evita mudanças inesperadas durante o processo.

Exemplo desenvolvimento de um site institucional com página inicial, página “Sobre”, blog e formulário de contato, utilizando HTML, CSS e JavaScript.

3. Backlog do Projeto

O backlog é uma lista organizada de todas as tarefas e funcionalidades que precisam ser desenvolvidas no projeto.

Ele permite:

* priorizar atividades;
* acompanhar o progresso do desenvolvimento;
* melhorar a organização da equipe.

Ferramentas como Kanban (ex.: Trello) podem ser usadas para visualizar as tarefas.

4. User Stories

As User Stories descrevem as funcionalidades do sistema a partir da visão do usuário.

Estrutura padrão:
“Como (tipo de usuário), eu quero (ação ou funcionalidade) para (benefício ou objetivo).”

Exemplo:
“Como visitante do site, eu quero acessar a página ‘Sobre’ para conhecer melhor a empresa.”

5. Wireframes

Os wireframes são esboços das telas do site que mostram a estrutura e organização dos elementos da página.

Eles ajudam a:

* planejar o layout;
* definir o fluxo de navegação;
* melhorar a experiência do usuário.

Podem ser:

* Baixa fidelidade:** simples, como um rascunho.
* Alta fidelidade:** mais detalhados e próximos do design final.

Ferramentas comuns: **Figma, Uizard e Octopus**.

6. Prototipação

A prototipação consiste na criação de versões iniciais do sistema para testar ideias e validar o projeto antes do desenvolvimento final.

Ela permite:

* visualizar o funcionamento do sistema;
* realizar melhorias antes da implementação;
* garantir uma melhor experiência para o usuário (UX).

Conclusão:
A definição de escopo, o planejamento do backlog, o uso de user stories, wireframes e protótipos são etapas essenciais no desenvolvimento de aplicações web, pois ajudam a organizar o projeto, alinhar expectativas e melhorar a experiência do usuário.
